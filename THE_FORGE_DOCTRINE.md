## THE FIVE IMMUTABLE LAWS

### Law 1: Truth Over Simulation
**"No Illusion" Rule**

- Never invent results or fake success
- No placeholders/mocks/stubs in production
- If you cannot verify it ran, it did not run
- Evidence is mandatory, not optional

**Manifestation:**
- BootForge: Refuses to write without typed confirmation
- Pandora Codex: Returns `manual_intervention_required` if validation fails
- Universal Legend Status: Real metrics, real traces, real health checks

### Law 2: Evidence Over Assumption
**"Audit-First" Mentality**

- Every action produces evidence
- Every operation creates an immutable record
- Every claim must be verifiable
- Every manifest is signed

**Manifestation:**
- BootForge: Evidence bundles with SHA-256 hashes
- Shadow Logger: Encrypted audit trails
- Observability: Correlation IDs, structured logs, traces

### Law 3: Control Over Convenience
**"Explicit Over Magic"**

- No hidden state
- No driver heuristics
- No automatic transitions
- No black boxes

**Manifestation:**
- Vulkan render graph: Explicit barriers, no guessing
- Job system: Deterministic scheduling, no blocking
- USB detection: Evidence-based, no assumptions

### Law 4: Offline Over Cloud
**"Sovereignty First"**

- Works without network
- Portable execution
- Self-contained artifacts
- No phone-home

**Manifestation:**
- BootForge USB: Runs from stick, writes evidence to stick
- Phoenix Key: Local authentication, no cloud dependency
- Bobby's Workshop: Offline-first workflows

### Law 5: Repairability Over Finality
**"Nothing Is Finished"**

- Systems can be examined
- Systems can be modified
- Systems can be rebuilt
- Systems leave receipts

**Manifestation:**
- BootForge: Can verify, can resume, can audit
- Reforge OS: Transparent layers, no hidden abstraction
- All projects: Deterministic, reproducible, debuggable
